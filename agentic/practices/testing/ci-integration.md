# CI Integration

**Category**: Engineering Practice  
**Applies To**: All OpenShift repositories  
**Last Updated**: 2026-04-08  

## Overview

OpenShift CI uses Prow for automated testing. All PRs must pass CI checks before merging.

## CI System Architecture

```
GitHub PR → Prow → ci-operator → Test Pod → Results
                     ↓
                 Build images
                 Create cluster
                 Run tests
                 Collect artifacts
```

## Job Types

| Job Type | Trigger | Purpose | Example |
|----------|---------|---------|---------|
| **presubmit** | On PR | Validate changes before merge | unit, integration, e2e |
| **postsubmit** | After merge | Build release artifacts | image builds |
| **periodic** | Scheduled | Continuous validation | upgrade tests, scale tests |

## Configuration Files

### ci-operator Config

Location: `openshift/release/ci-operator/config/openshift/<repo>/<repo>-<branch>.yaml`

```yaml
base_images:
  base:
    name: "4.16"
    namespace: ocp
    tag: base

build_root:
  image_stream_tag:
    name: release
    namespace: openshift
    tag: golang-1.21

images:
- dockerfile_path: Dockerfile
  to: machine-config-operator

tests:
- as: unit
  commands: make test-unit
  container:
    from: src

- as: verify
  commands: make verify
  container:
    from: src

- as: e2e-aws
  steps:
    cluster_profile: aws
    workflow: openshift-e2e-aws

- as: e2e-upgrade
  steps:
    cluster_profile: aws
    workflow: openshift-upgrade-aws
```

### Prow Job Config

Location: `openshift/release/ci-operator/jobs/openshift/<repo>/<repo>-<branch>-presubmits.yaml`

```yaml
presubmits:
  openshift/machine-config-operator:
  - name: pull-ci-openshift-machine-config-operator-master-unit
    always_run: true
    optional: false
    decorate: true
    path_alias: github.com/openshift/machine-config-operator
    spec:
      containers:
      - image: ci/src:latest
        command:
        - make
        - test-unit

  - name: pull-ci-openshift-machine-config-operator-master-e2e-aws
    always_run: true
    optional: false
    decorate: true
    spec:
      # Generated by ci-operator from config
```

## Common Job Patterns

### Unit Tests

```yaml
- as: unit
  commands: make test-unit
  container:
    from: src
```

**Runs**: On every PR  
**Timeout**: 1 hour  
**Required**: Yes  

### Integration Tests

```yaml
- as: integration
  commands: make test-integration
  container:
    from: src
```

**Runs**: On every PR  
**Timeout**: 2 hours  
**Required**: Yes  

### E2E Tests

```yaml
- as: e2e-aws
  steps:
    cluster_profile: aws
    workflow: openshift-e2e-aws
    test:
    - as: test
      commands: make test-e2e
      from: src
      resources:
        requests:
          cpu: 100m
          memory: 200Mi
```

**Runs**: On every PR  
**Timeout**: 4 hours  
**Required**: Yes  

### Upgrade Tests

```yaml
- as: e2e-upgrade
  steps:
    cluster_profile: aws
    workflow: openshift-upgrade-aws
```

**Runs**: Periodic (nightly)  
**Timeout**: 6 hours  
**Required**: No (informing)

|----------|---------|-------|

| `openshift-e2e-gcp` | E2E on GCP | Cloud-specific tests |

### Custom Workflow Steps

```yaml
- as: e2e-custom
  steps:
    cluster_profile: aws
    pre:
    - as: setup
      commands: ./hack/setup.sh
      from: src
    test:
    - as: test
      commands: make test-e2e
      from: src
    post:
    - as: gather
      commands: ./hack/gather-logs.sh
      from: src
```

## Debugging Failed Tests

### View Test Logs

```bash
# From PR, click "Details" on failed check
# Or use CLI:
prow-pr-logs <org>/<repo> <pr-number>
```

### Download Artifacts

```bash
# Artifacts stored in GCS bucket
# URL in PR comment from ci-operator
https://gcsweb-ci.apps.ci.l2s4.p1.openshiftapps.com/gcs/origin-ci-test/pr-logs/...
```

### Common Artifacts

```
artifacts/
├── build-log.txt           # Build output
├── junit/                  # JUnit test results
├── pods/                   # Pod logs
├── must-gather/            # Cluster diagnostics
└── e2e-test.log           # E2E test output

# Run same tests as CI
ci-operator --config=openshift/release/ci-operator/config/openshift/machine-config-operator/machine-config-operator-master.yaml --target=e2e-aws
```

## Best Practices

### 1. Keep Jobs Fast

**Target times**:
- Unit: <5 minutes
- Verify: <5 minutes
- Integration: <15 minutes
- E2E: <90 minutes

### 2. Use Appropriate Resources

```yaml
resources:
  requests:
    cpu: 100m      # Minimal for most tests
    memory: 200Mi
  limits:
    cpu: "2"       # Prevent runaway tests
    memory: 4Gi
```

### 3. Make Tests Reliable

- No sleeps - use Eventually
- Clean up resources
- Handle transient failures
- See [test-flake-policy.md](./test-flake-policy.md)

### 4. Fail Fast

```yaml
# Stop on first failure
- as: verify
  commands: make verify fmt vet
  # NOT: make verify; make fmt; make vet (continues on failure)
```

### 5. Cache Dependencies

```yaml
# Cache go modules
- as: unit
  commands: |
    export GOCACHE=/tmp/go-build-cache
    export GOMODCACHE=/tmp/go-mod-cache
    make test-unit
  container:
    from: src
```

## PR Workflow

### 1. Create PR

```bash
git push origin my-feature-branch
gh pr create
```

### 2. CI Triggers

Prow automatically:
- Runs presubmit jobs
- Comments status on PR
- Updates commit status checks

### 3. Monitor Results

```
✅ pull-ci-openshift-mco-master-unit
✅ pull-ci-openshift-mco-master-verify
⏳ pull-ci-openshift-mco-master-e2e-aws (running)
```

### 4. Address Failures

If tests fail:
1. Click "Details" to view logs
2. Download artifacts if needed
3. Fix issue and push new commit
4. CI re-runs automatically

### 5. Retest

```
/retest  # Re-run failed tests
/test <job-name>  # Run specific test
/test all  # Run all tests
```

### 6. Merge

Requirements:
- All required tests pass
- LGTM from reviewer
- Approved by approver
- No hold labels

## Optional vs Required Tests

```yaml
# Required (blocks merge)
- as: unit
  always_run: true
  optional: false

# Optional (informing only)
- as: e2e-upgrade
  always_run: false
  optional: true
  run_if_changed: "^(pkg/|cmd/)"
```

## Conditional Tests

```yaml
# Run only if specific files change
- as: e2e-metal
  run_if_changed: "^(pkg/daemon/|manifests/)"
  
# Skip if only docs changed
- as: e2e-aws
  skip_if_only_changed: "^docs/|^.*\\.md$"
```

## Prow Commands

Common `/` commands in PR comments:

| Command | Effect |
|---------|--------|
| `/retest` | Re-run failed tests |
| `/test <job>` | Run specific test |
| `/test all` | Run all tests |
| `/hold` | Prevent merge |
| `/hold cancel` | Remove hold |
| `/lgtm` | Mark as looks-good-to-me |
| `/approve` | Approve for merge |
| `/assign @user` | Assign reviewer |
